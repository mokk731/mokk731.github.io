在安装前确认以下环境满足要求：
    操作系统：Linux（主流发行版）、macOS、Windows WSL2、Android Termux
    Python：3.10+

    网络：需访问 GitHub 及选定的 LLM 提供商 API

macOS 安装
步骤 1：安装前置依赖
如果你使用的是全新的 Mac，需要先安装 Homebrew：
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
安装 Python 3.11+ 和 Node.js 18+：
brew install python@3.12 node

步骤 2：一键安装 Hermes Agent
curl -fsSL https://hermes-agent.org/install.sh | bash
安装脚本会自动：
    检查系统环境
    创建 Python 虚拟环境
    安装所有依赖包
    配置 CLI 命令

步骤 3：初始化配置
hermes init
交互式向导会引导你完成：
    选择模型提供商（推荐 OpenRouter 或 DeepSeek）
    输入 API Key
    选择默认模型
    配置基础权限

步骤 4：启动
hermes start
看到 Hermes Agent is running 就表示安装成功了。

Windows（WSL2）安装
步骤 1：启用 WSL2
在 PowerShell（管理员）中执行：
wsl --install -d Ubuntu-22.04
重启电脑后，打开 Ubuntu 终端。

步骤 2：在 WSL2 中安装
sudo apt update && sudo apt upgrade -y
sudo apt install -y python3.12 python3.12-venv nodejs npm git ripgrep
curl -fsSL https://hermes-agent.org/install.sh | bash

步骤 3-4：同上
hermes init
hermes start

验证安装
hermes --version    # 查看版本
hermes status       # 检查运行状态
hermes chat         # 进入交互式对话

FAQ：

Q1：安装脚本安全吗？
A：安装脚本完全开源，你可以在 GitHub 仓库 中查看源码。建议先用 curl -fsSL https://hermes-agent.org/install.sh | less 审查脚本内容。

Q2：安装后配置文件在哪里？
A：所有配置和数据存储在 ~/.hermes/ 目录下，包括 config.yaml、记忆数据库、技能文件等。

Q3：可以同时安装多个 Hermes Agent 实例吗？
A：同一台机器上通常只运行一个实例。如果需要多实例，建议使用 Docker 容器隔离。

我的安装过程：            ########

# 下载Docker
基于 Docker 部署 Hermes Agent 并接入飞书机器人
https://hub.docker.com/r/nousresearch/hermes-agent
docker pull nousresearch/hermes-agent:v2026.4.30

mkdir -p /srv/.hermes
chmod -R 755 /srv/.hermes

# 以交互方式启动容器以运行设置向导
docker run -it --rm \
  -v /srv/.hermes:/opt/data \
  nousresearch/hermes-agent:v2026.4.30 setup


https://hermesagent.org.cn/docs/getting-started/setup-wizard
Hermes Agent Setup Wizard

    执行 hermes setup 命令
    选择 Quick Setup
    当提示选择模型提供商时，选择 DeepSeek
    输入你的 DeepSeek API Key
    Base URL 填写 https://api.deepseek.com
    选择 deepseek-v4-flash 模型
    继续完成其余配置选项
    是否配置消息平台的界面 这里我们先先跑一个最小可用的版本出来，所以先跳过


  https://platform.deepseek.com  充值查账

  nano /srv/.hermes/config.yaml
  nano /srv/.hermes/.env


# 以网关模式运行
docker run -d \
  --name hermes \
  --restart unless-stopped \
  -v /srv/.hermes:/opt/data \
  nousresearch/hermes-agent:v2026.4.30 gateway run



# 对于持久化网关部署，使用 docker-compose.yaml 非常方便：

version: "3.8"
services:
  hermes:
    image: nousresearch/hermes-agent:v2026.4.30
    container_name: hermes
    restart: unless-stopped
    command: gateway run
    volumes:
      - /srv/.hermes:/opt/data
    # 取消注释以转发特定的环境变量，而不是使用 `.env` 文件：
    # 环境：
    #   - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
    #   - OPENAI_API_KEY=${OPENAI_API_KEY}
    #   - TELEGRAM_BOT_TOKEN=${TELEGRAM_BOT_TOKEN}
    deploy:
      resources:
        limits:
          memory: 4G
          cpus: "2.0"


# 交互式运行（CLI 聊天 ）                  ########
docker run -it --rm \
  -v /srv/.hermes:/opt/data \

# 选项 B：手动配置
将以下内容添加到 ~/.hermes/.env 文件中：
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=secret_xxx
FEISHU_DOMAIN=feishu
FEISHU_CONNECTION_MODE=websocket

# 可选但强烈推荐
FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy
FEISHU_HOME_CHANNEL=oc_xxx
